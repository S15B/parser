# ОТЧЕТ О ТЕСТИРОВАНИИ ПРОЕКТА "HTML ПАРСЕР"

## 1. ТЕСТОВЫЙ ПЛАН

Объект тестирования: HTML парсер 
Цель: Проверка извлечения изображений из HTML, поддержка локальных файлов, URL, Data URL

**Что проверяли:**
- Парсинг HTML и извлечение тегов `<img>`
- Сохранение локальных файлов
- Скачивание по URL
- Декодирование Data URL
- Поиск текста до/после изображений
- Обработка ошибок и граничных случаев


## 2. ПРОЕКТИРОВАНИЕ ТЕСТ-КЕЙСОВ

Разработано **14 тест-кейсов**, покрывающих основную функциональность:

 **Парсер** 
test_parse_text_only - Парсинг обычного текста 
test_parse_single_tag - Парсинг параграфа 
test_parse_img_tag - Извлечение изображения с alt 
test_parse_img_without_alt - Изображение без alt 
test_skip_script - Пропуск script тегов 
test_skip_comments - Пропуск комментариев 
 **Сохранение** 
test_save_data_url_png - Сохранение PNG из Data URL 
test_save_data_url_invalid - Некорректный Data URL 
test_copy_local_file - Копирование локального файла 
 test_copy_local_file_not_found - Обработка отсутствующего файла 
 **Граничные случаи** 
test_empty_html - Пустой HTML 
test_unclosed_tag - Незакрытый тег 
test_unicode_text - Unicode символы 
test_attributes_with_spaces - Пробелы вокруг знака =

## 3. ТЕХНОЛОГИЧЕСКИЙ СТЕК

В качестве основного языка используется **Python 3.9+** , так как именно на нем написан сам проект.

Для написания и запуска тестов используется встроенный фреймворк **unittest**. Его главное преимущество — он не требует дополнительной установки, так как входит в стандартную библиотеку Python.

Для работы с временными файлами и папками применяется модуль **tempfile** . Он позволяет автоматически создавать и удалять временные директории после завершения тестов, что исключает "засорение" файловой системы.

Запуск тестов осуществляется через командную строку, что обеспечивает простоту использования и возможность интеграции в автоматические системы сборки.
Для оформления отчетов выбран формат Markdown. Он удобен для чтения как в исходном виде, так и после преобразования в HTML или PDF.

**Преимущества выбранного стека:**
Не требует установки дополнительных библиотек
Работает на всех платформах (Windows, macOS, Linux)
Простой порог входа для новых участников
Легко интегрируется с системами CI/CD

## 4. РАЗРАБОТКА ЮНИТ-ТЕСТОВ

На основе разработанных тест-кейсов были созданы автоматизированные юнит-тесты.

Для удобства поддержки и запуска тесты были организованы в следующую структуру:

tests/
├── __init__.py                  # Маркер пакета Python
├── test_parser.py                # Тесты для парсера (6 тестов)
├── test_image_saving.py          # Тесты для сохранения изображений (4 теста)
└── test_edge_cases.py            # Тесты для граничных случаев (4 теста)

Для запуска всех тестов создан файл run_tests.py

Все разработанные юнит-тесты полностью покрывают тест-кейсы, описанные в разделе 2. Каждый тест-кейс реализован в виде отдельного тестового метода, что обеспечивает прослеживаемость между требованиями и кодом тестов.
